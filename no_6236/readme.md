## 문제 요약

N일 동안 사용할 금액이 주어질 때, 한 번에 K원을 인출해서 생활한다.
총 인출 횟수를 정확히 M번으로 맞추면서 K를 최소로 해야 한다.

중요한 조건:
- 돈이 모자라면 남은 돈을 넣고 다시 K원을 인출한다.
- 돈이 남아 있어도 원하면 다시 인출할 수 있다.

## 핵심 아이디어

정답 K의 범위를 먼저 잡는다.

- 최소값 후보(left): 하루 지출의 최대값
	- 어떤 날은 하루치 금액을 한 번에 써야 하므로 K는 이보다 작을 수 없다.
- 최대값 후보(right): 전체 지출 합
	- 이 값이면 한 번 인출로도 전체 기간을 버틸 수 있다.

이 범위에서 이분 탐색을 수행한다.

## 판정 함수: required_withdraw_count

주어진 K로 N일을 순서대로 시뮬레이션해서,
"필요한 최소 인출 횟수" cnt를 계산한다.

규칙:
- 시작 시 한 번 인출(count = 1)
- 현재 잔액으로 오늘 금액을 못 내면 새로 인출(count += 1)
- 그 뒤 오늘 금액을 차감

## 왜 cnt <= M 이 정답 조건인가

이 문제는 정확히 M번을 맞춰야 하지만,
최소 필요 인출 횟수가 M보다 작아도 중간에 일부러 추가 인출해서 M번을 맞출 수 있다.

따라서:
- cnt <= M 이면 가능한 K
- cnt > M 이면 불가능한 K

## 알고리즘 순서

1. 입력 받기
2. left = max(expenses), right = sum(expenses) 설정
3. while left <= right:
	 - mid = (left + right) // 2
	 - cnt = required_withdraw_count(mid, expenses)
	 - cnt <= M 이면 정답 후보 저장 후 right = mid - 1
	 - cnt > M 이면 left = mid + 1
4. 최종 정답 출력

## 복잡도

- 판정 함수 1회: O(N)
- 이분 탐색 횟수: O(log(sum(expenses)))
- 전체: O(N log(sum(expenses)))

N이 100,000이어도 충분히 통과 가능한 복잡도다.
