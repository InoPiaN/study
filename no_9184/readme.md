# BOJ 9184 - 신나는 함수 실행

## 풀이 아이디어

문제의 점화식을 그대로 재귀로만 호출하면 같은 상태를 계속 다시 계산해서 시간 초과가 난다.

핵심은 `(a, b, c)` 상태를 한 번 계산하면 저장해 두고 재사용하는 것(메모이제이션)이다.

- 유효한 상태 범위는 `0 ~ 20`만 보면 된다.
- `a > 20 or b > 20 or c > 20` 인 경우는 항상 `w(20, 20, 20)`으로 치환된다.
- 따라서 `21 x 21 x 21` 크기의 3차원 배열로 충분하다.

## 점화식 정리

1. `a <= 0 or b <= 0 or c <= 0` 이면 `1`
2. `a > 20 or b > 20 or c > 20` 이면 `w(20, 20, 20)`
3. `a < b < c` 이면
   `w(a, b, c-1) + w(a, b-1, c-1) - w(a, b-1, c)`
4. 그 외는
   `w(a-1, b, c) + w(a-1, b-1, c) + w(a-1, b, c-1) - w(a-1, b-1, c-1)`

## 현재 코드 방식

- `dp[a][b][c]`가 0이 아니면 이미 계산된 값이므로 바로 반환
- 아니면 점화식으로 계산 후 `dp[a][b][c]`에 저장
- 입력은 `-1 -1 -1`이 나올 때까지 반복
- 출력 형식은 문제 요구 그대로

## 복잡도

- 시간복잡도: `O(21^3)` (각 상태를 최대 1번 계산)
- 공간복잡도: `O(21^3)`

## 구현 포인트

- 파이썬에서도 재귀 호출 깊이는 이 문제 범위에서 충분함
- 외부 라이브러리 없이 3차원 리스트 메모이제이션으로 해결 가능
