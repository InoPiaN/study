# BOJ 12865 - 평범한 배낭

## 문제 요약
- 물건 `N`개가 있고, 각 물건은 `(무게 W, 가치 V)`를 가진다.
- 배낭이 버틸 수 있는 최대 무게는 `K`다.
- 각 물건은 **쪼갤 수 없고 0번 또는 1번만** 담을 수 있다.
- 담은 물건들의 가치 합의 최댓값을 구한다.

## 핵심 아이디어
이 문제는 대표적인 `0/1 Knapsack` 문제이며, DP로 해결한다.

- `max_value_by_capacity[c]`:
	허용 무게가 `c`일 때 만들 수 있는 최대 가치

물건을 하나씩 보면서, 각 무게 `c`에 대해 아래 두 경우를 비교한다.

1. 현재 물건을 넣지 않는다.
2. 현재 물건을 넣는다. (`남은 무게의 최적값 + 현재 물건 가치`)

즉,

`max_value_by_capacity[c] = max(기존값, max_value_by_capacity[c - item_weight] + item_value)`

## 왜 무게를 큰 쪽에서 작은 쪽으로 순회하나?
현재 물건을 한 번만 사용해야 하기 때문이다.

- 큰 쪽 -> 작은 쪽(역순)으로 갱신하면,
	같은 물건을 같은 라운드에서 중복 사용하지 않는다.
- 작은 쪽 -> 큰 쪽(정순)으로 갱신하면,
	방금 갱신된 값을 다시 사용해 같은 물건을 여러 번 쓴 효과가 난다.

## "이미 담긴 물건을 빼고 다른 조합으로 바꾸는" 경우도 고려되나?
고려된다.

코드가 실제로 물건을 빼고 넣는 시뮬레이션을 하지는 않지만,
각 용량에서 매번 `넣는 경우 / 안 넣는 경우`를 비교해서 최댓값으로 갱신한다.
그래서 기존 조합보다 더 좋은 조합(예: 1개 대신 2개 조합)이 나오면
DP 값이 자동으로 그 조합으로 교체된다.

## 복잡도
- 시간 복잡도: `O(N * K)`
- 공간 복잡도: `O(K)`
