# BOJ 1766 - 문제집

## 핵심 아이디어
- 선행 조건이 있는 순서 문제이므로 위상 정렬을 사용한다.
- 단, 여러 문제가 동시에 풀 수 있는 상태라면 번호가 작은 문제(쉬운 문제)부터 선택해야 한다.
- 이를 위해 `진입차수 0`인 정점 집합을 일반 큐가 아니라 최소 힙으로 관리한다.

## 입력 해석
- `A B`는 `A`를 먼저 풀어야 `B`를 풀기 좋다는 뜻이다.
- 방향 그래프로 보면 `A -> B` 간선이다.
- `B`의 진입차수(indegree)를 1 증가시킨다.

## 알고리즘
1. `adjacency_list`와 `indegree` 배열을 만든다.
2. 입력을 받을 때 바로 그래프와 진입차수를 구성한다.
3. 진입차수가 0인 모든 문제를 최소 힙 `available_problems`에 넣는다.
4. 힙에서 가장 작은 번호를 꺼내 정답 `solve_order`에 추가한다.
5. 꺼낸 문제에서 갈 수 있는 다음 문제들의 진입차수를 1씩 줄인다.
6. 진입차수가 0이 된 문제를 다시 힙에 넣는다.
7. 힙이 빌 때까지 반복한다.

## 정당성 요약
- 위상 정렬 특성상, 진입차수가 0인 문제만 선택하므로 선행 조건을 위반하지 않는다.
- 동시에 가능한 문제가 여러 개일 때 최소 힙에서 가장 작은 번호를 꺼내므로 "가능하면 쉬운 문제부터" 조건을 만족한다.

## 복잡도
- 간선 수를 `M`, 정점 수를 `N`이라 하면:
- 그래프 구성: `O(M)`
- 위상 정렬: 각 정점/간선을 한 번씩 처리 + 힙 연산
- 총 시간 복잡도: `O((N + M) log N)`
- 공간 복잡도: `O(N + M)`

## 구현 포인트
- 입력 크기가 크므로 `sys.stdin.readline`을 사용한다.
- 입력을 `arr` 같은 중간 리스트로 저장하지 않고, 읽는 즉시 `adjacency_list`와 `indegree`를 갱신해 메모리 사용을 줄였다.
- 변수명은 코테에서 읽기 쉬운 형태로 유지했다.
